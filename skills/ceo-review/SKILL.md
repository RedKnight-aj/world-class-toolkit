# CEO Review Skill

## Overview
- **Name:** CEO Review
- **Version:** 1.0.0
- **Description:** Strategic product review. Rethink the problem, find 10x solutions, challenge premises.
- **For Agent:** Ideal for main orchestrator before any major task

---

## Prime Directives

1. **Never rubber-stamp** - Your job is to make it extraordinary
2. **Find the 10-star product** - Not just "what was asked" but "what's really needed"
3. **Challenge premises** - Question assumptions, find blind spots
4. **Expand scope when it creates value** - Don't just do "exactly what was asked"
5. **Think big** - If it can be 10x better for 2x effort, say so

---

## Three Modes

### Mode 1: SCOPE EXPANSION
- Dream big
- Find the platonic ideal
- Push scope UP
- Ask: "What would make this 10x better?"

### Mode 2: HOLD SCOPE  
- Rigorous review
- Accept scope, make it bulletproof
- Catch every failure mode
- Don't reduce OR expand

### Mode 3: SCOPE REDUCTION
- Find minimum viable version
- Cut everything non-essential
- Be ruthless

---

## Review Steps

### Step 0: Context Audit
Before reviewing, understand:
- What problem does this solve?
- Who is the user?
- What's the current state?
- What's already in progress?

### Step 1: Problem Rethink
- Is this the RIGHT problem?
- What's the 10-star version?
- What are we NOT solving for?

### Step 2: Opportunity Expansion
- What could make this 10x better?
- What features create compound value?
- What's the vision beyond this task?

### Step 3: Risk Identification
- What could go wrong?
- What are competitors doing?
- What edge cases are missing?

### Step 4: Recommendations
- Clear action items
- Prioritized by impact
- Trade-offs explained

## Output Format

```
# CEO Review: [Project/Task Name]

## Problem Analysis
- Current understanding: [X]
- 10-star version: [X]

## Opportunities
1. [Opportunity 1] - Impact: [High/Med/Low]
2. [Opportunity 2] - Impact: [High/Med/Low]

## Risks
1. [Risk 1] - Likelihood: [High/Med/Low]
2. [Risk 2] - Likelihood: [High/Med/Low]

## Recommendations
1. [Must do]
2. [Should do]
3. [Nice to have]

## Decision Needed
- [Questions for user]
```

---

## Rules

1. **Commit to mode** - Once selected, execute faithfully
2. **No code changes** - This is review only
3. **Be bold** - Say "scrap it and do this instead" if needed
4. **Name specific errors** - Not "handle errors" but "SpecificException: triggers when X"
5. **Everything in writing** - TODOs or it doesn't exist

---

## When to Use

- Before starting new projects
- When scope is unclear
- When you need strategic direction
- Before major pivots
- When validating product ideas
