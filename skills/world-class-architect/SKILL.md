Use when planning new features, refactors, or system upgrades to ensure perfection and cost-efficiency.\n---\n\n# World-Class Architect\n\n## Core Philosophy\n**"Perfection is King."** Do not write code until the architecture is proven. Optimize for:\n1.  **Zero Waste:** No redundant API calls. Use local tools first.\n2.  **Determinism:** Use pipelines (Lobster) over chat loops.\n3.  **Resiliency:** Design for failure (Fallbacks, Auto-Healing).\n\n## Workflow\n\n### 1. The "ROI Audit" (Before you start)\n- **Ask:** "Does this add tangible value?"\n- **Check:** Can this be done with existing tools? (e.g. `apply_patch` instead of rewrite)\n- **Refine:** Simplify the design until it cannot be broken.\n\n### 2. Select the Pattern\n- **For Multi-Step Workflows:** Use **Lobster** (.lobster pipelines).\n- **For Data/Memory:** Use **LanceDB** (Vector Store).\n- **For Complex Research:** Spawn a **Researcher** sub-agent.\n- **For Massive Refactors:** Spawn a **Coder** sub-agent.\n\n### 3. The Blueprint\nCreate a `PLAN.md` before execution. Define:\n- **Objective:** Clear ROI goal.\n- **Constraints:** Rate limits, costs, security.\n- **Architecture:** The chosen pattern.\n- **Validation:** How will we prove it works?\n\n## References\n- **Patterns:** See [patterns.md](references/patterns.md) for 2026 Elite Architectures.\n
